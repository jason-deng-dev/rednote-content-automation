运营监控后台

running.moximoxi.net — 自动化系统统一管理平台
负责人：Jason Deng 日期：2026年3月 状态：规划中 — 三条管道部署完成后开始开发
背景与需求

平台目前运行三条自动化管道：XHS内容发布、RunJapan赛事数据抓取、乐天商品采集。管道运行状态目前只能通过服务器日志文件查看，这在日常运维和交接后均不可持续。

监控后台的目标是：让运营人员无需接触服务器、无需阅读日志，即可在浏览器中完整掌握所有管道的运行状态、配置参数，并在需要时手动触发操作。

核心功能模块
模块 01

首页 — 管道健康总览

  • 三条管道状态卡片（空闲 / 运行中 / 失败）
  • 最近一次运行时间与结果
  • 近7天/30天成功率
  • 下次计划运行时间
  • 异常时醒目标红提示
模块 02

XHS管道 — 全功能管理

  • 每日发帖计划配置（时间 + 内容类型）
  • 实时日志流（SSE推送）
  • 手动触发发帖（含预览模式）
  • XHS会话状态 + 一键重新登录
  • 历史发帖归档查看
  • Claude API Token用量追踪
模块 03

赛事抓取 — 数据监控

  • 本次抓取赛事数量（阈值告警 < 30）
  • 失败URL列表
  • 当前races.json数据预览
  • 历史运行记录
  • 手动触发抓取
模块 04

乐天管道 — 商品管理

  • 各分类商品数量（已缓存 / 已上架）
  • 定价配置编辑（利润率、运费、汇率）
  • WooCommerce导入日志与失败重试
  • 手动触发商品抓取
关键功能 — XHS会话重新登录

无需服务器访问的QR码登录流程

XHS会话每隔数周过期。后台提供一键登录流程：点击按钮后，服务器自动启动无头浏览器并导航至QR码登录页面，将截图以2秒间隔实时推送到后台页面。运营人员用手机扫描QR码，登录成功后会话自动保存，管道恢复正常运行。全程无需接触服务器或命令行。

技术实现

后台基于 Next.js 构建，前后端一体（App Router页面 + API路由），部署在同一台AWS Lightsail服务器上，与三条管道共用Docker环境。后台通过共享Volume读取各管道的状态文件，通过内部API向管道发送控制指令，无需额外服务器。

Feature Proposal

Operations Monitoring Dashboard

running.moximoxi.net — Unified management interface for all automation pipelines
Author: Jason Deng Date: March 2026 Status: Planning — development begins after all pipelines are deployed
Background & Purpose

The platform currently runs three automation pipelines: XHS content publishing, RunJapan race data scraping, and Rakuten product ingestion. Pipeline health can only be checked by reading raw server log files — which is not sustainable for day-to-day operations or after handoff.

The dashboard gives any operator full visibility and control over all three pipelines from a browser — no SSH, no terminal, no log files required.

Core Feature Modules
Module 01

Home — Pipeline Health Overview

  • Status card per pipeline (Idle / Running / Failed)
  • Last run timestamp and result
  • Success rate over last 7 / 30 days
  • Next scheduled run time
  • Prominent error alerts on failure
Module 02

XHS Pipeline — Full Management

  • Per-day posting schedule config (time + post type)
  • Live log stream (SSE real-time push)
  • Manual trigger with preview mode
  • XHS session status + one-click re-login
  • Published post archive viewer
  • Claude API token usage tracker
Module 03

Race Scraper — Data Monitoring

  • Races scraped per run (alert if < 30)
  • Failed URL list
  • Current races.json data preview
  • Full run history
  • Manual trigger
Module 04

Rakuten Pipeline — Product Management

  • Product counts per category (cached / live in store)
  • Pricing config editor (margin %, shipping, exchange rate)
  • WooCommerce import log + one-click retry on failures
  • Manual product fetch trigger
Key Feature — XHS Session Re-authentication

QR Code Login Flow — No Server Access Required

XHS sessions expire every few weeks. The dashboard exposes a one-click re-login flow: clicking the button triggers the server to launch a headless browser, auto-navigate to the QR code login screen, and stream screenshots to the dashboard at 2-second intervals. The operator scans the QR code with their phone. Login is detected automatically — the session is saved and the pipeline resumes. No server access or command line required at any point.

Technical Implementation

Built on Next.js (App Router pages + API routes in one codebase), deployed on the same AWS Lightsail instance as all three pipelines. The dashboard reads pipeline state from a shared Docker volume and sends control commands to pipelines via internal APIs — no additional server infrastructure required.
